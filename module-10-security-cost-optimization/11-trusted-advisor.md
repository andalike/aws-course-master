### Automation with CloudWatch Events

```yaml
AWSTemplateFormatVersion: '2010-09-09'
Description: 'Trusted Advisor Automation'

Resources:
  # SNS Topic for Alerts
  TrustedAdvisorAlertTopic:
    Type: AWS::SNS::Topic
    Properties:
      TopicName: trusted-advisor-alerts

  # EventBridge Rule for Trusted Advisor status changes
  TrustedAdvisorRule:
    Type: AWS::Events::Rule
    Properties:
      Name: 'TrustedAdvisorCheckStatusChange'
      Description: 'Capture Trusted Advisor check status changes'
      EventPattern:
        source:
          - aws.trustedadvisor
        detail-type:
          - Trusted Advisor Check Item Refresh Notification
        detail:
          status:
            - ERROR
            - WARN
      State: ENABLED
      Targets:
        - Id: SNSTarget
          Arn: !Ref TrustedAdvisorAlertTopic
        - Id: LambdaTarget
          Arn: !GetAtt TrustedAdvisorHandler.Arn

  # Lambda function to process Trusted Advisor findings
  TrustedAdvisorHandler:
    Type: AWS::Lambda::Function
    Properties:
      FunctionName: trusted-advisor-handler
      Runtime: python3.11
      Handler: index.handler
      Role: !GetAtt LambdaRole.Arn
      Timeout: 60
      Code:
        ZipFile: |
          import json
          import boto3
          import os

          sns = boto3.client('sns')

          def handler(event, context):
              print(f"Received event: {json.dumps(event)}")

              check_name = event['detail'].get('check-name', 'Unknown')
              status = event['detail'].get('status', 'Unknown')
              check_id = event['detail'].get('check-item-detail', {})

              # Format notification
              message = f"""
              Trusted Advisor Alert

              Check: {check_name}
              Status: {status}
              Details: {json.dumps(check_id, indent=2)}

              Action Required: Please review this finding in the AWS Console.
              """

              # Send notification
              sns.publish(
                  TopicArn=os.environ['SNS_TOPIC_ARN'],
                  Subject=f'Trusted Advisor Alert: {status} - {check_name}',
                  Message=message
              )

              return {'statusCode': 200}
      Environment:
        Variables:
          SNS_TOPIC_ARN: !Ref TrustedAdvisorAlertTopic

  # Lambda permission for EventBridge
  LambdaInvokePermission:
    Type: AWS::Lambda::Permission
    Properties:
      FunctionName: !Ref TrustedAdvisorHandler
      Action: lambda:InvokeFunction
      Principal: events.amazonaws.com
      SourceArn: !GetAtt TrustedAdvisorRule.Arn

  # IAM Role for Lambda
  LambdaRole:
    Type: AWS::IAM::Role
    Properties:
      AssumeRolePolicyDocument:
        Version: '2012-10-17'
        Statement:
          - Effect: Allow
            Principal:
              Service: lambda.amazonaws.com
            Action: sts:AssumeRole
      ManagedPolicyArns:
        - arn:aws:iam::aws:policy/service-role/AWSLambdaBasicExecutionRole
      Policies:
        - PolicyName: SNSPublish
          PolicyDocument:
            Version: '2012-10-17'
            Statement:
              - Effect: Allow
                Action: sns:Publish
                Resource: !Ref TrustedAdvisorAlertTopic
```

## Hands-on: Trusted Advisor Audit

```bash
#!/bin/bash
# Trusted Advisor Audit Script (requires Business/Enterprise support)

echo "=== AWS Trusted Advisor Audit ==="
echo "Date: $(date)"
echo ""

# Note: This requires Business or Enterprise support
# For Basic/Developer, use the console

# Get all check IDs
echo "Fetching Trusted Advisor checks..."
CHECKS=$(aws support describe-trusted-advisor-checks --language en --query 'checks[].{id:id,name:name,category:category}' --output json)

# Security checks
echo -e "\n=== Security Checks ==="
echo $CHECKS | jq -r '.[] | select(.category=="security") | "\(.name): \(.id)"'

# Cost optimization checks
echo -e "\n=== Cost Optimization Checks ==="
echo $CHECKS | jq -r '.[] | select(.category=="cost_optimizing") | "\(.name): \(.id)"'

# Get summaries for key checks
echo -e "\n=== Check Results ==="

# S3 Bucket Permissions
echo -e "\nS3 Bucket Permissions:"
aws support describe-trusted-advisor-check-result \
    --check-id "Qch7DwouX1" \
    --query 'result.{Status:status,FlaggedResources:flaggedResources[].metadata}' \
    --output json 2>/dev/null || echo "Check not available (requires Business support)"

# Security Groups - Unrestricted Access
echo -e "\nSecurity Groups - Specific Ports Unrestricted:"
aws support describe-trusted-advisor-check-result \
    --check-id "HCP4007jGY" \
    --query 'result.{Status:status,FlaggedResources:flaggedResources[].metadata}' \
    --output json 2>/dev/null || echo "Check not available (requires Business support)"

# MFA on Root Account
echo -e "\nMFA on Root Account:"
aws support describe-trusted-advisor-check-result \
    --check-id "7DAFEmoDos" \
    --query 'result.status' \
    --output text 2>/dev/null || echo "Check not available"

# Low Utilization EC2
echo -e "\nLow Utilization EC2 Instances:"
aws support describe-trusted-advisor-check-result \
    --check-id "Qch7DwouX1" \
    --query 'result.{Status:status,ResourcesSummary:resourcesSummary}' \
    --output json 2>/dev/null || echo "Check not available (requires Business support)"

echo -e "\n=== Audit Complete ==="
```

## Summary

AWS Trusted Advisor is a powerful tool for maintaining AWS best practices:

1. **Cost Optimization**: Identify waste and savings opportunities
2. **Performance**: Ensure optimal resource utilization
3. **Security**: Detect vulnerabilities and misconfigurations
4. **Fault Tolerance**: Improve resilience and reliability
5. **Service Limits**: Prevent throttling before it happens

**Key Takeaways**:
- Available checks depend on your support plan
- Prioritize security RED items immediately
- Automate alerts with EventBridge
- Review recommendations regularly
- Document exceptions and track improvements over time
- Consider Business support for full access to all checks
